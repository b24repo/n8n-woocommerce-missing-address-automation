QUICK START (5 MINUTES)
================================================================================

STEP 1: Import the Workflow
   → Open your n8n instance
   → Go to Workflows → Import from File
   → Select workflow.json from this folder
   → Click Import

STEP 2: Add Credentials
   → In n8n, go to Settings → Credentials
   → Create 3 credentials:
      1. Claude API (get key: https://console.anthropic.com/)
      2. WooCommerce (REST API credentials from WordPress)
      3. Google Sheets (service account JSON)

STEP 3: Set Environment Variables
   → On your n8n server, add these:
      WOOCOMMERCE_DOMAIN=yourstore.com
      GOOGLE_SHEETS_TRACKING_ID=YOUR_SHEET_ID
      SLACK_WEBHOOK_URL=https://hooks.slack.com/...
      SITE_URL=https://yourstore.com

STEP 4: Create Tracking Sheet
   → Create new Google Sheet
   → Add headers: Timestamp | Order ID | Order Number | Customer Name | 
     Customer Email | Customer Phone | Missing Fields | Action Taken | 
     Status | Follow-up Date
   → Share with Google service account email

STEP 5: Test It
   → Send test webhook request:
   curl -X POST https://your-n8n/webhook/woocommerce-missing-address \
     -H "Content-Type: application/json" \
     -d '{"body": "Order #12345 missing apartment number", "subject": "Alert"}'

STEP 6: Activate
   → Toggle workflow to "Active"
   → You're live!

Total time: ~30 minutes

================================================================================
WHAT THIS WORKFLOW DOES
================================================================================

When a 3PL warehouse sends a missing address notification:

1. RECEIVES the webhook notification
2. EXTRACTS order/customer data using Claude AI
3. VALIDATES the order exists in WooCommerce
4. ANALYZES which address fields are missing
5. SENDS automated customer email requesting address correction
6. SENDS SMS reminder (optional, if phone number available)
7. ALERTS fulfillment team on Slack
8. LOGS everything in Google Sheets for tracking

All with comprehensive error handling and alerting.

================================================================================
CREDENTIALS YOU NEED TO GATHER
================================================================================

1. CLAUDE API KEY
   Source: https://console.anthropic.com/account/api-keys
   Format: sk-ant-...
   Where used: Claude AI extraction node

2. WOOCOMMERCE REST API CREDENTIALS
   Source: WordPress Admin → WooCommerce → Settings → Advanced → REST API
   What you need: Consumer Key + Consumer Secret
   Format: Basic Auth

3. GOOGLE SHEETS SERVICE ACCOUNT
   Source: https://console.cloud.google.com/
   What you need: JSON key file with credentials
   Format: JSON credential file

4. GOOGLE SHEETS TRACKING SHEET ID
   Create: New Google Sheet
   Source: URL bar: https://docs.google.com/spreadsheets/d/{SHEET_ID}/...
   Extract: The SHEET_ID part
   Add headers: See QUICKSTART.md

5. (OPTIONAL) SLACK WEBHOOK URL
   Source: Slack workspace settings → Apps → Webhooks
   Format: https://hooks.slack.com/services/...

6. (OPTIONAL) TWILIO CREDENTIALS
   Source: Twilio account console
   Format: Service ID + Integration ID

WORKFLOW OVERVIEW
================================================================================

The workflow contains 17 nodes organized into 5 categories:

ENTRY POINT (1 node):
   └─ Webhook: Receives notifications from 3PL system

PROCESSING (4 nodes):
   ├─ Sanitize incoming data
   ├─ Claude AI extraction
   ├─ Address analysis
   └─ Data formatting

DECISIONS (3 nodes):
   ├─ Order number valid?
   ├─ Order found in WooCommerce?
   └─ Address issues exist?

ACTIONS (5 nodes):
   ├─ Email customer
   ├─ SMS customer (optional)
   ├─ Slack team alert
   └─ Google Sheets logging

ALERTS (4 nodes):
   ├─ Error notifications
   ├─ Order not found alert
   ├─ False positive notification
   └─ Critical error handler
